# Seats 测试文档

## 测试概述

本项目包含完整的单元测试，覆盖所有核心功能模块。测试使用纯JavaScript编写，不依赖任何测试框架，可以独立运行。

**总计: 109 个测试用例，全部通过**

## 测试文件结构

```
__tests__/
├── test-stores.test.js    # SeatGridState 状态管理测试 (41个用例)
├── test-export.test.js    # 导出功能核心逻辑测试 (36个用例)
├── test-i18n.test.js      # 国际化功能测试 (32个用例)
├── run-all-tests.js       # 测试运行器（运行所有测试）
└── README.md              # 本文档
```

## 运行测试

### 运行所有测试

```bash
node __tests__/run-all-tests.js
```

### 运行单个测试文件

```bash
# SeatGridState 状态管理测试
node __tests__/test-stores.test.js

# 导出功能测试
node __tests__/test-export.test.js

# 国际化测试
node __tests__/test-i18n.test.js
```

### 通过 npm 脚本运行（可选）

在 `package.json` 中添加：

```json
{
	"scripts": {
		"test": "node __tests__/run-all-tests.js"
	}
}
```

然后运行：

```bash
npm test
```

## 测试设计原则

### 1. 自包含

每个测试文件都是独立的，不依赖外部测试框架或库。必要的类（如 `SvelteSet`、`SeatGridState`）都在测试文件中模拟。

### 2. 清晰的测试结构

每个测试文件使用统一的测试运行器模式：

```javascript
function runTests(suiteName, tests) {
	// 运行测试并输出结果
}

const tests = [
	{
		name: '测试名称',
		fn: () => {
			// 测试逻辑
			return true; // 通过
			// 或返回错误消息：'失败原因'
		}
	}
];
```

### 3. 完整的覆盖

- **正常情况**: 验证功能按预期工作
- **边界情况**: 空数据、满数据、单行/列数据
- **错误情况**: 无效输入、超出范围、资源不足
- **组合场景**: 多个功能同时使用

### 4. 格式化的输出

测试结果输出清晰易读：

```
============================================================
测试套件: SeatGridState 状态管理
============================================================

--- 初始化 ---
  ✓ 初始化10x10空表格
  ✓ 初始化时过道列为空

...

------------------------------------------------------------
结果: 41 通过, 0 失败
------------------------------------------------------------
```

### 5. 退出码

- 所有测试通过：退出码 0
- 任何测试失败：退出码 1

这使得测试可以在CI/CD流程中使用。

## 添加新测试

1. 在对应测试文件的 `tests` 数组中添加新测试用例
2. 使用描述性的测试名称
3. 测试函数应返回 `true`（通过）或错误消息（失败）
4. 运行测试验证新用例

示例：

```javascript
{
    name: '新的测试用例',
    fn: () => {
        const grid = new SeatGridState();
        // 执行操作
        grid.fillCell(0, 0, '张三', () => {}, () => {});
        // 验证结果
        if (grid.tableData[0][0] !== '张三') {
            return '单元格应填入张三';
        }
        return true;
    }
}
```

## 注意事项

1. **测试是纯JavaScript**：不依赖Svelte运行时，可以直接用Node.js运行
2. **模拟类是简化版**：测试中的 `SeatGridState` 是源码的简化实现，不包含Svelte的响应式特性（`$state`）
3. **导出测试仅验证逻辑**：不生成实际的Excel文件，只验证行映射、列偏移等核心计算逻辑
4. **i18n测试模拟浏览器环境**：使用全局变量模拟 `navigator.language` 和 `localStorage`

## 持续集成

测试可以集成到CI/CD流程中：

```yaml
# .github/workflows/test.yml
name: Tests
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
      - run: node __tests__/run-all-tests.js
```
